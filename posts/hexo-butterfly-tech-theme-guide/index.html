Hexo Butterfly 深色科技感博客主题改造技术记录

发表于2026-05-25|更新于2026-05-25|技术实践
|浏览量:

这篇文章记录本站从 Hexo 默认风格切换到 Butterfly 主题，并进一步设计为深色科技感个人技术博客的实现过程。

改造的目标不是堆叠炫酷动画，而是在保证阅读体验的基础上，让首页体现计算机视觉、三维重建、神经渲染和机器人感知方向的个人研究气质。

项目目标

本站基于 Hexo 构建并部署到 GitHub Pages：

  • 博客名称：湘云的博客
  • 副标题：科研、技术与生活记录
  • 作者：Chen Xiangyun
  • 主题：Butterfly 5.5.4
  • 部署地址：https://Chenxiangyunyun.github.io

视觉设计希望实现：

  1. 深蓝、黑色与紫色构成的暗色背景；
  2. 具有研究方向识别度的首页 Hero；
  3. 低密度星空与节点连线效果；
  4. 简洁清晰的玻璃拟态文章卡片；
  5. 可见但克制的鼠标交互反馈；
  6. 移动端与低动画偏好的正常适配。

深色科技风配置

站点默认使用暗色模式，同时保留主题自带切换按钮：

首页 Banner 不使用大尺寸图片，而是通过多层渐变构建光场：

选择渐变背景有三个原因：

  • 文件体积小，加载速度快；
  • 深蓝与紫色光场容易形成科技感；
  • 后续调色只需修改配置，不必重新制作图片资源。

首页 Hero 设计

首页 Hero 以研究身份为视觉中心，包含：

  • RESEARCH SYSTEM / ONLINE 状态标识；
  • 博客名称与副标题；
  • 首页说明文字；
  • 六个研究方向标签；
  • 椭圆轨道线与发光节点。

研究方向展示为：

  • Computer Vision
  • 3D Reconstruction
  • Neural Rendering
  • Gaussian Splatting
  • Point Cloud
  • Robotics Perception

这些附加元素通过 source/js/tech-home.js 在首页动态加入，而不是修改 Butterfly 的模板文件。这种方式可以降低主题升级时的合并成本。

动态星空实现

首页背景使用一个仅作用于 Hero 的 Canvas 层。实现思路如下：

  1. 创建数量受限的随机星点；
  2. 让星点以极低速度漂移；
  3. 对距离较近的星点绘制透明连线；
  4. 为下半区额外分配星点，使效果覆盖到 Banner 底部；
  5. 鼠标靠近时增强局部星点亮度和连线强度。

  • 星空只运行于首页 Hero，不进入正文；

标题与标签交互

首页标题使用浅色到蓝紫色的渐变文字。鼠标悬停时：

  • 渐变颜色缓慢流动；
  • 文字轻微上移和放大；
  • 增加柔和辉光；
  • 轨道节点同步增强亮度。

介绍语和研究标签也拥有轻量反馈：

  • 介绍语悬停时文字提亮；
  • 研究标签悬停时轻微上浮；
  • 标签边框与背景产生冷色光效。

这些效果聚焦于用户主动交互状态，不会持续干扰阅读。

文章卡片交互

首页文章列表保留 Butterfly 原生渲染逻辑，只通过独立 CSS 调整表现。

卡片整体

文章卡片采用暗色半透明玻璃效果：

  • 半透明背景与细边框；
  • 顶部短光线标识；
  • 悬停时向上浮动；
  • 边框与周围光晕增强；
  • 表面经过一次柔和扫描光；
  • 点击按下时出现短暂回落与内发光。

文章标题

标题不再仅是静态文本。悬停标题或进入卡片时：

  • 标题显示蓝紫流光；
  • 直接悬停标题时文字轻微右移；
  • 标题下方展开渐变线条；
  • 辉光让标题与分类链接的交互层次保持一致。

文章摘要

摘要文段增加独立反馈：

  • 鼠标进入摘要或卡片时文字提亮；
  • 摘要区域轻微右移；
  • 左侧出现蓝紫色发光竖线。

因此卡片内的标题、摘要、分类标签都具备一致而清晰的交互表现。

总结

本次改造以 Butterfly 原生配置为基础，通过独立 CSS 与 JavaScript 增加了适度的科研科技感：首页具有深色光场、动态星空、研究标签与标题流光，文章卡片提供清晰的交互反馈，同时保留 Hexo 的内容组织、部署方式和主题回退能力。

对于个人技术博客而言，首页视觉应服务于身份表达和内容浏览。比起大量复杂动画，更重要的是形成稳定、可维护、可长期阅读的整体风格。
